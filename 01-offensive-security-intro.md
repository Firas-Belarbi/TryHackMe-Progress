## ðŸ”‘ Key Concepts Learned

### 1. Ethical Hacking Fundamentals
- Ethical hackers always work with **permission** and follow a defined **scope**
- The goal is to discover security weaknesses *before* real attackers do  
- Reporting and documenting findings is an essential part of the job  

### 2. Attack Surface Understanding
- Websites have multiple potential entry points:  
  - Login pages  
  - Input fields  
  - URLs  
  - Hidden directories  
- A larger attack surface = more opportunities for exploitation  

### 3. Common Web Weaknesses (High-Level Overview)
The room briefly introduced examples such as:  
- Weak authentication  
- Missing input validation  
- Misconfigured services  
- Exposed admin panels  

*(Deep exploitation like SQLi, XSS, LFI, etc. comes in future rooms.)*

### 4. Attacker Mindset
- Gather information first (**reconnaissance**)  
- Test different inputs and observe behavior  
- Think â€œWhat happens if I break the rules?â€  
- Be patient, methodical, and creative
